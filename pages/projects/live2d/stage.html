OpenSeeFace runs locally. Get it: github.com/emilianavt/OpenSeeFace. Run the tracker + osf_bridge.js (serves its socket on :8080 by default), then turn tracking on. This must match the bridge's HTTP_PORT, and the page must be on http (a hosted https page can't reach a local socket).

tracking rig
This is the tracking — each row sends an input → a model parameter, with input range → output range (like VTube Studio's parameter list). Loaded with the standard defaults; if the auto-rig got one wrong, repoint the param or change the ranges (e.g. out −1 → 1, or swap them to invert). Idle cover that should hide when tracking? add faceFound → that param, in 0→1, out 1→0.

Browser Source: add this page's URL, right-click the source → Interact to load your model, turn the backdrop OFF, press H to hide this panel.

Camera won't turn on in OBS? OBS's browser blocks camera access — there's no allow-popup and no camera picker, so webcam tracking is denied. Two ways around it:
• Window-Capture a normal browser tab running this page instead (the real browser shows the camera prompt + picker), or
• use OpenSeeFace locally: download the local copy above into your repo clone, get OpenSeeFace, run its tracker + osf_bridge.js, open the page from the bridge at http://localhost, then pick tracking source OpenSeeFace. (A hosted https page can't reach a local socket — that's why the local copy is needed.)
